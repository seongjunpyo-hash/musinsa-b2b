# MUSINSA Global B2B — 핸드오프 패키지

## 1. 개요

MUSINSA Global B2B 플랫폼의 디자인/UX 시안 (단일 HTML 프로토타입). 백엔드 API 연동 및 운영 환경 구축을 위한 명세 자료입니다.

해외 파트너(바이어)들이 무신사 카탈로그를 보고 발주(B2B 도매 주문)를 진행하고, 무신사 어드민이 주문을 검토/승인/반려하는 플로우입니다.

## 2. 화면 구성 (2개 앱)

### A. 어드민 콘솔 (`b2b-admin-console.html`)
무신사 내부 운영팀용. 5개 페이지로 구성:

| 페이지 | 주요 기능 |
|---|---|
| 대시보드 | KPI 카드(상품/주문/금액/파트너), 인기 상품 TOP 7, 최근 주문, 수주회 상태, 환율 설정 |
| 상품 관리 | 상품 CRUD, 일괄 노출/비노출, 일괄 정보 수정/삭제, 엑셀 일괄 업로드/다운로드, 파트너별 홀세일가 보기 |
| 전시 설정 | 카테고리별 노출 순서, NEW/HERO 상품 선정, 메인 배너 4종 + 스페셜 배너 관리 |
| 주문 조회 | 주문 필터(상태/파트너/기간/검색), 일괄 승인/반려, 단일/일괄 엑셀 다운로드, 주문 상세, 반려 사유 입력 |
| 파트너 관리 | 파트너 등록/수정/삭제, 파트너별 할인율 설정, 활성/비활성(로그인 차단) |

### B. 파트너 포털 (`b2b-partner-portal.html`)
파트너(바이어) 발주용:

| 페이지 | 주요 기능 |
|---|---|
| 메인 | 카테고리별 상품 노출, NEW/HERO 섹션, 5개 컬렉션 배너 |
| 카테고리 | 카테고리 필터, 시즌/연도 필터, 상품 그리드 |
| 상품 상세 | 모달, 컬러 스왓치 + 사이즈 선택, 수량 입력 |
| 장바구니(Bag) | 담은 상품 확인 |
| Order Confirmation | 주문 검토, 사이즈/수량 편집, 일괄 삭제, 엑셀 업로드/다운로드, 주문 제출 |
| My Orders | 본인 주문 내역, 상세 보기, 주문 취소, 반려 사유 확인, Admin 제안 주문 |

## 3. 핵심 비즈니스 룰 (요약)

자세한 내용은 [`BUSINESS_RULES.md`](./BUSINESS_RULES.md) 참고.

- **홀세일가 계산**: `Math.floor((retail_price_krw / exchange_rate) × (1 - discount_rate) × 100) / 100`
- **주문 상태 전이**: 주문접수 → (승인) 주문완료 / (반려) 반려 / (취소) 주문취소
- **파트너 할인율**: 1~100 사이 정수 (%), 파트너별로 다름
- **상품 노출**: `is_visible: true` 만 파트너 포털에 표시
- **파트너 비활성화**: `is_active: false` 시 파트너 로그인 차단
- **반려 시 Admin 제안**: 주문 반려 시 수정안(`suggested_items`) 함께 제시 가능 → 파트너가 그 제안을 한 번에 다시 주문 가능

## 5. 현재 프론트엔드 구조

### 기술 스택 (시안)
- 단일 HTML 파일 + React 18 + Babel (CDN)
- 데이터 저장: 브라우저 `localStorage` (탭 간 동기화는 `storage` 이벤트)
- 엑셀 처리: SheetJS (CDN)
- 빌드 시스템 없음, 그냥 HTML 파일 더블클릭으로 실행

### 운영 단계에서 변경 필요한 부분
| 영역 | 현재 (시안) | 운영 권장 |
|---|---|---|
| 빌드 | Babel CDN | Vite/Next.js 등 번들러 |
| 데이터 | localStorage | REST API + DB |
| 동기화 | localStorage storage 이벤트 | WebSocket / polling |
| 이미지 | base64 또는 CDN URL | S3/Cloudinary 등 별도 호스팅 |
| 인증 | 클라이언트 단순 비교 | JWT/세션 기반 인증 + 권한 분리 |
| 환경 | 하드코딩 | dev/staging/production 분리 |

### 코드 진입점

**어드민** (`b2b-admin-console.html`)
- `App` 컴포넌트 (라인 ~6018) — 라우팅, 상태 관리
- 데이터 영속화: 라인 6088~6097 (`localStorage.setItem`)
- 페이지 컴포넌트: `Dashboard`, `ProductsPage`, `DisplaySettingsPage`, `OrdersPage`, `PartnersPage`

**파트너** (`b2b-partner-portal.html`)
- `App` 컴포넌트 (라인 ~4580)
- 카탈로그/장바구니/주문 흐름이 같은 컴포넌트 안에서 페이지 전환

## 6. 인증 / 권한

### 어드민 로그인
- 단일 계정 (`admin@musinsa.com` / `musinsa.com`)
- SHA-256 해시 비교 (시안 수준, 실서비스 부적합)

### 파트너 로그인
- 등록된 파트너 이메일만 허용
- `is_active: false` 파트너는 로그인 차단
- 비밀번호는 시안에서 미구현 (운영에서 추가 필요)

### 데이터 권한
- 어드민: 모든 데이터 (R/W)
- 파트너: 본인 주문만 R/W, 카탈로그 R, 본인 정보 R
- **운영에서 반드시**: 백엔드에서 토큰 기반 권한 검증 (현재 시안은 클라이언트만 검증)

## 7. 주요 화면 / UX 결정

- **어드민 최소 해상도**: 1440px (그 이하는 테이블 가로 스크롤)
- **파트너 포털**: 모바일 대응 (반응형)
- **테이블 디자인**: 좌측 컬럼 sticky-left (가로 스크롤 시 핵심 정보 고정)
- **Sticky thead**: JS 기반 shadow thead로 가로 스크롤 + 세로 sticky 양립
- **파트너 주문 상세**: (상품, 컬러) 단위로 행 머지, 사이즈만 분리 표시

## 8. 디자인 시스템

- **폰트**: Pretendard Variable (한글), system-ui (영문)
- **컬러 팔레트**:
  - 메인: `#111` (블랙), `#fff` (화이트)
  - 보조 텍스트: `#666`, `#888`, `#999`
  - 보더: `#e5e5e5`, `#d1d5db`, `#eee`
  - 강조: `#2563eb` (블루, 액센트), `#dc2626` (레드, 에러/반려)
  - 상태 배지: 주문접수=옅은 블루, 주문완료=옅은 그린, 반려=옅은 레드, 주문취소=그레이
- **Border-radius**: 4~8px (대부분 4px)
- **Spacing**: 8px 그리드 (8/16/24/32...)
